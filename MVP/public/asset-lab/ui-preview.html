Reference-grounded pixel UI for the tactical layer · 2026-07-10 · three generation tests, ~$0.30 total
Verdict: it works. PixelLab can generate our combat UI — right-click menus, health/status plates, and style-matched icon sets — in one consistent gunmetal-and-amber pixel language that sits naturally next to our existing characters and props. Icon sets can be grounded on our actual game sprites via style references (16 icons per call). UI panels take text + palette hints (no image refs), but land on-style anyway.

What we learned (the fine print)

  • Style refs are real and they work — on objects/icons (style_images, up to 8), 8-direction objects, and tiles. Not on create_ui_asset panels — those take text + palette hints only, but the "gunmetal + amber" prompt language lands on-style reliably.
  • Base64 refs must never pass through chat — a single flipped character corrupts the PNG. New pipeline script MVP/scripts/asset-pipeline/pixellab-mcp.mjs calls the PixelLab MCP over HTTP and inlines files from disk (@file:path), byte-perfect. This is now the batch tool for icon/asset runs.
  • Small canvas = multi-candidate packs — at 64 px one call yields 16 icons (each its own description), at 32 px up to 64. Review → keep the good ones → each becomes its own asset.
  • Pixel fonts exist too — create_font outputs a real .ttf; one call would give the laptop/combat UI a matching typeface. Not yet tested (next battery).
